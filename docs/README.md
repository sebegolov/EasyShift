## docs

Документация и контекст разработки EasyShift.

Основные разделы:

- `context` — архитектурные решения и общая стратегия (например, `data-storage-strategy.md`).
- `api` — план и описание HTTP‑эндпоинтов (MVP и дальнейшее развитие).
- `flows` — пользовательские сценарии и диалоги в боте/мини‑приложении.
- `finance` — монетизация, подписки и ограничения (планы по PVZ и т.д.).
- `security` — требования по 152‑ФЗ, защита данных, роли и доступы.
- `setup` — инструкции по развёртыванию (локально и на прод‑сервере).

Эти файлы — основной источник правды для другой нейросети или нового разработчика.
