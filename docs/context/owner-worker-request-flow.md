# Сценарий: запрос работника на ПВЗ (главный тест)

## Обзор

1. **Работодатель** создаёт смену по выбранному ПВЗ и публикует её.
2. **Система** находит работников, в чью зону попадает этот ПВЗ, и создаёт им уведомления `shift_published`.
3. **Работники** откликаются (создают `ShiftApplication`); могут снять отклик до выбора работодателя.
4. **Работодатель** в отдельной таблице видит отклики (только со статусом `submitted`), рейтинг работника; выбирает одного.
5. **Выбранному** приходит уведомление `shift_assigned` с контактом работодателя; **остальным** — `shift_slot_taken`.
6. При **снятии отклика** работником он пропадает из списка у работодателя (статус `withdrawn`).

---

## API для меню работодателя

- **Список ПВЗ:** `GET /pvz?ownerId={ownerId}` → массив ПВЗ.
- **Создать смену:** `POST /shifts` — тело: `pvzId`, `ownerId`, `title`, `description?`, `startAt`, `endAt`, `baseRate`, `currency?`, `responseDeadlineAt?`, `confirmationDeadlineAt?`, `urgencyLevel?`.
- **Опубликовать смену (запросить работников):** `POST /shifts/:id/publish` — по смене вычисляются работники по зонам (ПВЗ в радиусе зоны), создаются события `shift_published` для их User id.
- **Список смен работодателя:** `GET /shifts/owner/:ownerId` — смены с `assignment`.
- **Список откликов по смене:** `GET /shifts/:id/applications` — только статус `submitted`, в каждом объекте есть `workerProfile` (рейтинг, количество смен, имя пользователя). Отозванные отклики не возвращаются.
- **Назначить работника:** `POST /shifts/:id/assign` — тело: `applicationId`, `workerId`, `agreedRate`. После этого создаются уведомления `shift_assigned` (выбранному, с контактом) и `shift_slot_taken` (остальным откликнувшимся).

---

## API для работника

- **Откликнуться:** `POST /shifts/:id/applications` — тело: `workerId`, `proposedRate?`, `message?`, `isOutOfZoneOffer?`.
- **Снять отклик:** `POST /shifts/:shiftId/applications/:applicationId/withdraw` — тело: `{ "workerId": "..." }`. Доступно пока смена в статусе `collecting_responses`; после снятия работник пропадает из списка откликов у работодателя.

---

## Уведомления (NotificationEvent)

- **shift_published** — смена опубликована, ПВЗ в зоне работника; `payload: { shiftId }`.
- **shift_assigned** — вас выбрали на смену; `payload: { shiftId, ownerContactName, ownerContactPhone }`.
- **shift_slot_taken** — место по смене занято; `payload: { shiftId }`.

Обработчик (бот или воркер) по `eventType` и `userId` может отправить сообщение в Telegram и т.п.

---

## Матчинг по зонам

При `POST /shifts/:id/publish` вызывается `MatchingService.findCandidateWorkerIds(shiftId)`:

- Берётся смена и её ПВЗ (lat, lng).
- Выбираются активные зоны работников (`WorkerZone`, `isActive: true`), у которых `verificationStatus !== 'restricted'`.
- Для каждой зоны считается расстояние (Haversine) от центра зоны до точки ПВЗ; если расстояние ≤ `radiusKm`, в кандидаты добавляется **User id** этого работника (не WorkerProfile id).
- Уведомления создаются для этих User id (канал `telegram`, тип `shift_published`).

---

## Отзывы о работнике

Сейчас в ответе `GET /shifts/:id/applications` по каждому отклику приходит `workerProfile` с полями: `rating`, `completedShiftsCount`, `reliabilityScore`, имя из `user.fullName`. Отдельной модели отзывов (reviews) пока нет; при необходимости её можно добавить и вывести на экран «карточка работника» для работодателя.

## Mini App (меню работодателя и работника)

Исходники: `apps/mini-app/src/`. Сборка: `npm run build` в `apps/mini-app`, выход — `dist/`.

- **Работодатель:** главная → «Меню работодателя» → список смен, кнопка «Запросить работника» (выбор ПВЗ, даты, ставка, создание и публикация) → по смене в статусе «сбор откликов» — «Кто откликнулся» (таблица: имя, рейтинг, смены, кнопка «Выбрать»). После выбора одного выбранному уходит уведомление с контактом, остальным — «место занято».
- **Работник:** главная → «Меню работника» → блок «Мои отклики» (кнопка «Снять отклик» по каждому), блок «Уведомления» (смены в зоне — «Откликнуться»; сообщения о назначении/место занято).

Для работы с API при сборке задайте `VITE_API_URL` или на странице задайте `window.__API_BASE__` (URL бэкенда).
